- Android APK build passes with the current submodule and staged pdockerd.
- Offline compatibility audit passes with 55 PASS, 0 FAIL, 0 SKIP.
- UI action wiring audit passes, including daemon readiness wait, legacy builder
  env, detached compose up, terminal palette, editor tools, and image deep-link.
- Android Wi-Fi ADB quick smoke passes on `100.95.153.249:45033`: pdockerd
  starts through the debug activity hook, `docker version` negotiates with
  API 1.43, and bundled `docker compose version` reports v2.35.1.
- Android full smoke currently fails at Dockerfile `RUN` because the bundled
  PRoot runner cannot exec the Ubuntu rootfs on-device (`Bad address` /
  signal 11). The Build API now returns a Docker error frame so the CLI exits
  non-zero instead of letting compose continue with a missing image.
- Backend Dockerfile build smoke passes for:
  - a small single-RUN Dockerfile
  - a multi-step RUN/COPY/RUN Dockerfile
- Backend compose smoke passes for `docker compose up -d --build`, `compose ps`,
  and `compose down`.
- Backend build now emits layer snapshot progress so it no longer looks frozen
  during expensive rootfs snapshotting.
